# Estructura del Proyecto Frontend

El proyecto frontend sigue una estructura modular orientada a la separación de responsabilidades, práctica recomendada en proyectos de ingeniería de software modernos.

La organización principal del directorio `src` se define de la siguiente manera:

- pages/: Contiene las vistas principales del sistema. Cada subcarpeta representa una página accesible mediante el sistema de rutas.
- components/: Agrupa componentes reutilizables utilizados en múltiples páginas.
- context/: Destinado a la gestión de estado global de la aplicación.
- services/: Preparado para la comunicación con servicios externos o backend.
- hooks/: Almacena hooks personalizados para encapsular lógica reutilizable.
- styles/: Contiene los estilos globales y específicos del sistema.

Esta estructura mejora la legibilidad del código, facilita el mantenimiento y permite que nuevos desarrolladores comprendan rápidamente la organización general del proyecto.
